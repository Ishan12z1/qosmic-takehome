# CRO Audit: graza

Run ID: `graza_20260613_4c589e`  
Date: 2026-06-13

## Executive Summary

Graza’s biggest CRO problem is not checkout friction or missing trust. The shopping journey completed cleanly through cart and checkout entry with a 5.0/5 friction score, and the sampled Sizzle PDP already shows strong review proof and a substantial review module (`artifacts/graza_20260613_4c589e/pages/shopping_journey.json`, `artifacts/graza_20260613_4c589e/screenshots/shopping_journey_cart.png`, `artifacts/graza_20260613_4c589e/screenshots/shopping_journey_checkout.png`, `artifacts/graza_20260613_4c589e/screenshots/product_page_1_products_sizzle.png`). The sharper issue is interpretive friction: the brand’s playful naming and rich visual storytelling are memorable, but first-time shoppers still have to decode what Sizzle, Drizzle, Frizzle, refill cans, sprays, duos, and trios mean in practice before they know which product is right for them (`artifacts/graza_20260613_4c589e/screenshots/homepage_0_home.png`, `artifacts/graza_20260613_4c589e/screenshots/product_page_1_products_sizzle.png`, `artifacts/graza_20260613_4c589e/screenshots/collection_page_6_collections_bundles.png`).

The next major issue is that Graza’s expansion logic is stronger in assortment than in the sampled journey. Bundles, gifts, refills, and subscriptions clearly exist across navigation, collections, and FAQ content, but the captured cart drawer for a small spray order is narrow and does not strongly escalate that shopper into a bigger pantry-building path (`artifacts/graza_20260613_4c589e/screenshots/collection_page_6_collections_bundles.png`, `artifacts/graza_20260613_4c589e/screenshots/shopping_journey_cart.png`, `artifacts/graza_20260613_4c589e/screenshots/faq_shipping_returns_8_pages_faqs.png`). Technical health also matters here: mobile and desktop page speed both failed, OG tags were missing sitewide in the sample, and alt coverage is weak, which means the brand’s rich visual system is likely costing it both speed and shareability (`artifacts/graza_20260613_4c589e/technical_checks.json`).

## Proposed Experiments

### EXP-01: Translate whimsical product names into clearer first-buy decisions

| Field | Value |
|---|---|
| Pillar | Conversion |
| Surface | Homepage hero, first-scroll product modules, and collection entry |
| URL | https://graza.co/ |
| Evidence | artifacts/graza_20260613_4c589e/screenshots/homepage_0_home.png |
| Hypothesis | Making first-buy decisions more explicit in plain cooking language will improve homepage-to-PDP click-through because new shoppers currently have to decode Graza’s playful product naming and format system before they understand what belongs in their kitchen. |
| Primary change | Add a decision layer such as “For everyday cooking,” “For finishing,” “For high-heat spray,” and “For stocking up,” mapped directly to Sizzle, Drizzle, Frizzle, and bundles. |
| Primary KPI | Homepage-to-PDP click-through rate |
| Decision rule | Ship if homepage-to-PDP click-through improves by at least 8% relative without reducing session depth over 3 weeks. |
| Expected lift | +8-14% |
| Confidence | 84% |
